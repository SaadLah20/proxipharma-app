0) PROMPT RACINE (“DESIGN LANGUAGE”) — À LANCER EN PREMIER
================================================================================
Tu es designer produit pour une app web santé / officine au Maroc. Propose une
direction visuelle unique et cohérente pour toute une plateforme (patient +
pharmacien + annuaire public). Contraintes : mobile-first ; accessibilité
(contrastes WCAG AA) ; bilinguisme FR/AR (RTL possible) ; ton chaleureux et
clair, pas hospitalier ; accents discrets santé/pharma (sans croix médicale
stéréotypée omniprésente). Définis pour livrables : palette (primaire, secondaire,
succès/avertissement/erreur/info), typo (titres + corps, lisibles petite taille),
rayons cartes/chips/radius, styles boutons primaire-secondaire-lien-ghost, champs
formulaire états défaut-focus-erreur, ombres, espacements 4/8 multiples, grille.
Inclus 3 ambiances variations (premium sobre, approachable friendly, très minimal)
et une matrice tokens type design system léger pour dev (noms FR optionnels).

1) NAVIGATION & HEADER (patient / pharmacien / public)
================================================================================
Design d’un bandeau fixe top mobile : logo texte discret, menu hamburger ou onglets
selon rôle, icône notifications avec pastille, avatar ou initiales. Sur desktop :
barre plus large avec mêmes éléments. Style ProxiPharma — confiance locale — CTA
secondaire “mon espace”. États : scroll compressé ; notification non lues ; RTL
avec miroitement logique sans casser UX.

Variante mood image (sans texte UI lisible détaillé) :
“premium mobile healthcare app chrome, soft green and warm white, subtle shadow,
Marocco-friendly modern SaaS navbar, dribbble trending UI moodboard, minimalist”

2) LISTE DEMANDES — ESPACE PATIENT (hub cartes demandes produits / ordonnances)
================================================================================
Écran mobile : titre “Mes demandes”, filtres chips (À traiter, En cours,
Terminées, Annulées), barre recherche facultative sous les filtres, liste de cartes.
Chaque carte : réf publique courte (ex. D042/26), nom officine ou avatar,
statut lisible avec couleur pastel, lignes métier KPI (ex. lignes préparées,
annulées comptoir), chevron ou chemin visuel swipe-friendly. Vide : illustration
minimaliste + CTA explorer pharmacies. Accessible, grands zones touch.

Ajoute une variante tableau compact pour breakpoint desktop (colonnes réf /
pharmacie / statut / date).

3) LISTE DEMANDES — ESPACE PHARMACIEN (“file d’attente production”)
================================================================================
Dashboard liste des demandes entrantes traitées par équipe officine ; stress sur
priorité (< 15 min) et volumétrie. Cartes avec code demande patient visible,
pastille urgente SLA, snippets patient (sans photo obligatoire), statut étape
workflow. Filtres : nouvelles, en cours, retard, récence. Indicateur KPI header
(nb en attente, > 24h). Style outil métier précis mais calme pas alert fatigue ;
microcopy FR court professionnel pharmacien.

4) DÉTAIL DEMANDE — PATIENT (validation produits / alternatives / suivi confirmé)
================================================================================
Flux long-scroll mobile : bloc contexte officine avec contact WhatsApp/icône téléphone ;
section produits lignes comme cartes imbriquées avec : nom médicamenteux + dosage,
pastille disponibilité, quantité stepper ou lecture seule, alternatives radio max 3 ;
état désactivé “rupture marché” grisé avec message empathique pas culpabilité ;
deux niveaux après validation officine (“ce que vous avez validé” vs “préparation
actuelle”) en deux surfaces visuellement distinctes (fond différent léger timeline
ou colonne lecture seule) ; bloc historique type journal avec lignes événements.
CTA primaire évident en bas sticky quand pertinent. Mention accessibilité : focus ring.
